# docs/simulation-pipeline.md

本文基于 `js/game.js` 中的 `nextYear()`，说明“推进一年”时的执行顺序与每一步职责。

## 年度推进总览（nextYear）

1. **重置年度一次性开关**
   - 重置政策和交易类 `used` 标记（如再分配、商税、盐/布贸易、官盐投放、土地开垦本年标记）。
   - 目的：确保“每年一次”动作在新年度可重新使用。

2. **年份 +1 并清空事件修正器**
   - `world.year += 1`
   - 调用 `clearEventModifiers(world)` 清空各阶层 `eventModifier`。
   - 目的：把短期冲击限定在本年度内，避免跨年累积污染。

3. **机构自动建立检查（前置达成即建制）**
   - 依次检查并确保政府、警务、卫生、法院、税务、贸易、工程机构可用。
   - 目的：使制度层随条件自动进入可运转状态。

4. **同步机构配置输入**
   - 对官员人数、工资、税率、信用评级、贸易和工程参数进行“数值清洗与约束”。
   - 目的：把 UI 输入转成安全、可计算的状态值。

5. **预结算机构成本与薪资总额**
   - 计算政府工资账单。
   - 预计算警务/卫生/法院/税务/贸易/工程年度成本，并汇总到 `totalSalaryCost`。
   - 目的：在经济主循环前拿到财政刚性支出基线。

6. **教育年度结算（settleEducationYear）**
   - 调用 `processEducationYear(world)` 更新在校、毕业、资格池。
   - 结算商办学费流入商人收入池。
   - 结算官办教育支出与“学子下乡”支出（优先粮劵，不足再扣粮）。
   - 目的：先完成教育的人口与财政影响，再进入全年经济计算。

7. **经济主更新（updateEconomy）**
   - 统一推进农业、产业、市场、货币、价格、危机触发、行为惩罚、外交联动等经济变量。
   - 返回当年经济结果（产出、税收、损失、事件消息等）。

8. **人口主更新（updatePopulation）**
   - 根据当年经济/社会状态更新人口规模与结构。

9. **科研推进（updateResearch）**
   - 推进当前技术剩余年限，若完成则返回完成项并触发后续加成/日志。

10. **机构效果二次应用（按机构逐项）**
    - 警务：重新计算效果，应用商业修正、生活质量事件修正、稳定度变化、成本更新。
    - 卫生：更新健康成本与低健康年计数，必要时记录提示。
    - 法院：应用商业修正与税务/生活质量修正，更新成本。
    - 税务/贸易/工程：更新各自效果与成本；贸易效果会写回商业相关指标。
    - 目的：把“制度执行效果”在经济年度内最终落地。

11. **信用评级上限约束**
    - 根据法院效果给出的评级上限，必要时下调 `creditRating`。

12. **在建水渠完工结算**
    - 遍历 `pendingCanals`，将到期项目转入完工：增加耕地、提升基础亩产、更新数量并写日志。

13. **公共卫生灾害判定**
    - 若连续低健康达到阈值，触发瘟疫：人口一次性下降并重算人口结构，记录事件。

14. **特殊行为/短板提示**
    - 如警务纸张不足、人才不足等提示写入年志。

15. **地主低满意导致开垦阻断**
    - 若地主满意度过低，阻断待生效开垦增地（清空对应待生效值）。

16. **记录经济快照与年度摘要**
    - `recordEconomySnapshot(...)` 写入 `economyHistory`（最多保留 20 条）。
    - `logYearSummary(...)` 生成该年核心摘要文本。

17. **追加细分结算日志**
    - 追加教育、财政、警务、卫生、贸易、工程等条目化日志。
    - 若经济模块返回信用危机/行为消息/外交消息，也逐条写入。
    - 若科技完成，写入科技完成日志。

18. **年末生活质量与满意度最终计算**
    - `calculateLifeQuality(world)`：先算生活质量。
    - `calculateClassSatisfaction(world)`：再把生活质量与事件修正合成为四阶层满意度。
    - 目的：统一在年末进行“最终社会状态”收敛。

19. **钱庄与债务年终结算**
    - 先同步钱庄容量限制 `syncMoneylenderCaps(world)`。
    - 若已解锁钱庄系统：
      - `processGovernmentDebtYear(world)` 结算利息与还款；
      - `finalizeMoneylenderYear(world, interestDue)` 结算民间放贷、税收、可能新增商铺；
      - 写入债务与钱庄经营日志及惩罚消息。

20. **自动保存与渲染**
    - `saveGame(state, { auto: true })` 自动存档。
    - `render()` 刷新全部 UI。

---

## 关键顺序设计意图

- **先清空事件修正，再计算全年**：保证“年度事件”不跨年叠加。
- **教育结算早于经济主更新**：教育财政与人才产出会立即影响当年经济和社会。
- **生活质量/满意度在年末统一收敛**：避免多个模块在中途反复覆盖满意度。
- **债务与钱庄放在尾部**：把“年度经营结果”统一作为年终金融清算输入。
- **最后自动保存**：确保存档对应的是完整结算后的状态。
